# Contributing Guidelines

Thanks for contributing to `quiz-game-c`.

## Getting Started

1. Fork the repository
2. Create a feature branch
3. Build locally with `make`
4. Test your changes manually in the terminal
5. Open a pull request with a clear summary

## Coding Style

- Use C11-compatible code
- Keep functions small and focused
- Prefer descriptive variable names
- Update documentation when behavior changes
- Run formatting if `clang-format` is available

## Pull Request Checklist

- [ ] The project builds successfully
- [ ] I tested the main flow locally
- [ ] I updated docs if needed
- [ ] I kept changes scoped and readable

## Suggestions for Good First Issues

- Add more quiz categories
- Add timer-based mode
- Improve leaderboard formatting
- Add tests for storage behavior
